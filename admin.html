GitHub connection
Where to get a token:
  1. Go to github.com/settings/personal-access-tokens/new
  2. Set Repository access to "Only select repositories" → choose this repo.
  3. Under Permissions → Repository permissions, set "Contents" to Read and write.
  4. Generate the token and paste it above. Treat it like a password — anyone with it can edit the site.
This page is public (the repo is public), so the password screen only keeps casual visitors out — the token is what actually protects saving. Don't share your token, and revoke it on GitHub if you ever suspect it leaked.

Migrate existing media to Cloudinary

Any photo/video you uploaded before switching to Cloudinary is still sitting in your GitHub assets/ folder (paths like assets/news/xxx.png). Click the button below to pull every one of those in, upload it to Cloudinary, and rewrite the link automatically. New links are only stored in memory until you click Save changes at the bottom afterwards — nothing is deleted from GitHub, so this is safe to run more than once.

Academy — Courses

Add and edit the courses and lessons students see under the Meatika Course tab on /academy/. This uses the same GitHub connection as the rest of this page (set it up in GitHub connection above and click Load current content first) — it reads and writes a separate file, academy/academy-data.json, so it has its own Load/Save buttons below rather than using the main Save changes button at the bottom of the page.

Academy — Approved Students

Controls who can get past the Google sign-in screen on /academy/. Anyone can sign in with Google, but only emails you've approved here actually reach the course dashboard — everyone else sees a "pending approval" screen. This uses its own connection (Firebase, not GitHub) and saves immediately — there's no separate Save button and it doesn't touch data.json.

Sign in with the Google account you used to create the Academy's Firebase project, to manage the approved list. (One-time Firebase setup is in academy/README-academy.md — do that first if you haven't yet.)

Academy — Credit Top-ups

Students buy credits by paying via ABA/Bakong transfer (details are set in academy/academy.js, near PAYMENT_INFO) and submitting their transaction reference in the Academy app. Nothing is credited automatically — review each request below against your bank/e-wallet statement, then Approve or Reject. Approving adds the credits straight to that student's balance; the student sees the result next time they open the Academy. This uses the same Firebase connection as the Approved Students panel above.

Sign in above (Academy — Approved Students) to manage credit top-ups too — both panels share the same sign-in.

Sections & cards

Pages

Create standalone pages with their own title, intro text, content, and sections/cards — useful when a card should open more detail inside the site instead of an external link. Each page works exactly like the homepage above. Once a page exists, open any card below and set its Link type to "Internal page" to point to it.

Each page also has a Page content area where you can add photo, video, and text blocks in any order and any mix — use it alone for a simple content page (leave sections/cards empty), or combine it with sections/cards below for a linkable card grid too.

Photos and videos on pages upload straight to Cloudinary the moment you choose the file — no extra save step needed for the media itself. Click Save changes below afterwards to save the page text and publish it (which also records the new Cloudinary links).

Learning Center tags

These are the content tags Learning Center posts can be filed under — they show up as the filter tabs on the public Learning Center page. Add a new one, rename an existing tag's English and/or Khmer label, or delete one you no longer need. If you delete a tag that posts are still using, those posts are automatically moved to another tag instead of being left unset.

Learning Center content

Write education posts — articles or videos — with a cover photo, an optional video (paste a YouTube/Facebook/Vimeo link, or upload a short video file), and text in English and Khmer. Pick a Content tag below — this controls which tab the post shows up under on the public Learning Center page. Manage the list of available tags in Learning Center tags above. Each post also gets a Share this article row (Facebook, Telegram, X, WhatsApp, copy link) automatically — nothing to configure. Uncheck Published to keep working on a draft without it showing on the public news.html list (a draft is still viewable if you open its direct link, so you can preview it first).

This admin panel uses one shared password for the whole team, not separate per-person logins — anyone with the password can post any content type. To credit who wrote a post, fill in Posted by on that post; it's just a label, not a login.
